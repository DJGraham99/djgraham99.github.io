Northumberland Rock Art Project

https://github.com/DJGraham99/NorthumberlandRockArtProject.git

Summary

This project was completed for my Computer Science MSc dissertation at Newcastle University. The project involved retrieving data from an out-of-date SQL database and then using that data to create a responsive website that linked directly to the data. There are several main sections which encompass the clients desires for the website which will be used as subheadings in the following explanation.

User search

The search section was designed so that it could be usable for all ranges of user technological ability whilst also having a large selection of search capabilities for more advanced users.

Search results

In order to best incorporate the needs of the client as well as ensure that the website design was suited to as wide a range of target audience as possible, the search results were designed to be shown in three distinct formats: record view, list view and map view.
The record view was created utilising Bootstrap card formats while the map view used Leaflet to display a base map and then layer the geospatial information on top.

Individual Record Page

These individual pages are dynamically created from the recorded information in the database. This requires multiple complex MySQL queries to extract the data and then the use of PHP to then process and display the data.

User Functionality

In order to provide a service for the widest audience possible. The ability to save panels was made available for users with and without an account. This was done by utilising JavaScript local storage for users with no account, and a dedicated user database for those with an account.

Maps and Navigation

This was implemented as an additional functionality as it is not core to the client's specification. These maps utilise Leaflet and openRouteMap Service API to give walking directions to users to the rock art panels and any locations stored within the default Nominatim geocoder.

Browse

The browse function was implemented to allow users to browse panels if they were less certain of the specific details of the panel. The current browsing methods are browsing by panel name and by the type of marking at the site.

Gallery and Images

A modal was used for all images on the site so that when an image is clicked upon, a JavaScript function utilises AJAX to gather details about the image and then display the image and details larger for the user to view.

Cavallo Team Project

Cavallo Link

Summary

This project was completed as part of my Computer Science MSc. I completed this project in a group with several others so will only mention the parts of the project that I completed. The whole code is available on GitHub, but it is stated in the code comments who completed which aspects. This project involved creating a website and database for an example food delivery company. The database would need to be able to store product information and prices, with the capability for users to make an account which would also store their personal details. The physical website would need to have a section for customers and employees. The customer section would allow users to create accounts and browse items, and place orders. The employee section would involve a section to all employees to monitor the quantity of products sold and their types. This would also involve creating a delivery system which would assign drivers deliveries and then allow the drivers to check off when they are completed. The following information is dividied into the sections that I completed.

Order Confirmation

The order confirmation page confirms to the customer the details of their order. To create a personal and intimate experience for the user, the page utilises the user's account details to customise the page. This utilises session variables for the user to then execute MySQL database queries to retrieve both the user and their order details. The page then alternates based on whether the order is for collection or delivery. For collection orders the page displays the user's name, the estimated time when the order will be ready to collect, a summary of the order and a unique order ID. For delivery orders the page shows the customer's name, the customer's delivery address, estimated delivery time, order summary and a unique order ID.

Delivery Driver System

This page shows all the orders that are currently out for delivery. Each order is assigned to a driver and the driver can search for their driver code and then mark the delivery as complete which will then mark the time of completion in the order database.

Account creation / Login / Logout

This allows users to input their details in order to create an account and be able to login. There are several forms of validation set up to ensure data entered is valid. Additional data cleaning is completed to minimise the risk of SQL injections.

Checkout Page

This page was worked on by several people due to this being where many of the website's sections intertwine. My role within this page was to autofill the data boxes with the users account information whilst also allowing the data to be changed for that particular order. All inputted information is validated and filtered. I was then involved with working with people who specialised on the mock payment system to utilise the information and send it to the API so that it could be further processed.
